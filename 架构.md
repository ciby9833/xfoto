
一、系统总体架构

[ 用户 ]
   ↓（触摸屏操作）
[ Electron EXE（主程序） ]
   ↓
├── UI（React）
├── Flow Controller（流程控制核心）
├── Camera Adapter（调用 digiCamControl）
├── Image Engine（美颜/滤镜）
├── Layout Engine（拼图/相框）
├── Print Service（打印）
├── Payment Service（支付轮询）
├── Local DB（SQLite）
   ↓
[ 本地文件系统（照片/拼图结果） ]
（可选）
   ↓
[ 云端 API（订单/图片/模板） ]

⸻

二、模块职责（每一块干什么）

1️⃣ Flow Controller（最核心）✅ 已完成

相当于"导演"，控制整个流程：

待机 → 支付 → 拍照 → 选图 → 拼图 → 打印 → 完成

实现：src/core/FlowController.js
- EventEmitter 状态机，9 个状态，合法跳转表
- transition(nextState, data) / reset() / getSessionData()

⸻

2️⃣ Camera Adapter ✅ 代码完成 / ⚠️ 真机需 Windows 验证

调用 digiCamControl

职责：
* 触发拍照 → 返回图片路径
* Mock 模式：循环返回 data/captures/sample1~3.jpg

实现：src/modules/camera/CameraAdapter.js
待办：data/captures/ 放入 3 张测试图才能跑通 mock 流程

⸻

3️⃣ Image Engine ⏳ Phase 3

处理：
* 滤镜（MVP只做这个）
* 美颜（后面再加）

技术：Canvas / OpenCV / MediaPipe
实现：src/modules/image/ImageEngine.js（当前为 stub）

⸻

4️⃣ Layout Engine ✅ 代码完成 / ⚠️ 需 sample 图片才能验证

做：
* 相框叠加
* 拼图（2×6条）

实现：
- 浏览器 Canvas 合成（RenderingPage.jsx）→ base64
- 主进程写磁盘（src/modules/layout/LayoutEngine.js）

待办：assets/frames/ 放入相框 PNG 才能验证相框叠加

⸻

5️⃣ Print Service ✅ 代码完成 / ⚠️ 真机需 Windows 验证

做：
* 调用 SumatraPDF 命令行打印
* 控制尺寸

实现：src/modules/print/PrintService.js

⸻

6️⃣ Payment Service ⏳ Phase 2

做：

实现：src/modules/payment/PaymentService.js（当前为 stub，mock 立即返回成功）

7️⃣ Local DB（SQLite）✅ 已完成

三、用户交互流程

⸻

✅ 标准流程

① 待机页（吸引用户）          ✅ IdlePage.jsx
↓
② 点击开始                    ✅
↓
③ 选择模板（相框）             ✅ SelectTemplatePage.jsx
↓
④ 显示支付二维码               ✅ PaymentPage.jsx（mock 立即成功）
↓
⑤ 支付成功                    ✅
↓
⑥ 倒计时拍照（3张）            ✅ ShootingPage.jsx（需 sample 图）
↓
⑦ 选照片（或自动）             ✅ PreviewPage.jsx
↓
⑧ 拼图（加相框）               ✅ RenderingPage.jsx（Canvas合成）
↓
⑨ 打印                        ✅ PrintingPage.jsx（Windows 验证）
↓
⑩ 显示下载二维码               ⏳ DonePage 暂无二维码，8秒后返回
↓
返回待机

⸻

六、你开发时最容易犯的错（帮你避坑）

❌ 错误1：先做美颜 → 先跑通流程
❌ 错误2：先做后台 → MVP不需要
❌ 错误3：先做复杂UI → 先功能
❌ 错误4：自己写相机SDK → 先用 digiCamControl

⸻

七、最终你会得到什么系统

用户体验：扫码 → 拍照 → 出片
技术系统：Electron + digiCamControl + Canvas拼图 + Windows打印

⸻
总结：

先打通"物理世界"（相机 + 打印）
再补"软件体验"（UI + 美颜）
